# NotebookLM 調査依頼スキル
Claude Code が直接アクセスできない情報の調査が必要なとき、ユーザーに NotebookLM での調査を依頼する。自動化はしない。ユーザーが「開く → 貼る → コピーして戻す」だけで完結するように、すべての情報を1メッセージにまとめて提示する。

## 発動条件
以下に該当する調査が必要なとき:
- **YouTube 動画** — 動画内の発言・議論の分析
- **ポッドキャスト / 音声ファイル** — 音声コンテンツの内容把握
- **50ページ以上の PDF / 書籍** — 長文ドキュメントの横断分析
- **Google Drive 内ファイル群** — スプレッドシート、スライド等の一括分析
- **50個以上のソースの横断調査** — ノートブックに蓄積されたソースの横断分析

またはユーザーが「NotebookLM で調べて」と明示した場合。

## 依頼パッケージの出力
1メッセージにまとめて提示する。できるだけ短く、手順が伝わる最小限にする。

**重要: このスキルの出力フォーマットルール（このスキル内のみ適用）**
- プロンプトは引用ブロック付きコードブロック（`> ```...````）で囲む（コピーボタン自動表示 + 視覚的に区別）
- コードブロック内では `##` 見出し間やテーブル前後に空行を入れない

> 悪い例:
> ```
> ## 業績データ
> | 指標 | 値 |
> |---|---|
> | 売上 | 100億 |
>
> ## リスク要因
>
> ## 要約
> ```

> 良い例:
> ```
> ## 業績データ
> | 指標 | 値 |
> |---|---|
> | 売上 | 100億 |
> ## リスク要因
> ## 要約
> ```

### 構成
依頼パッケージは以下の3パートで構成する:

**1. URL**
`https://notebooklm.google.com` と、新規/既存ノートブックの指示。

**2. 調査プロンプト**
調査内容に応じて Claude Code が最適な方式を選び、方式の説明とプロンプトをまとめて提示する:
- **URL ソース追加** — YouTube URL 等をソースとして追加。複数候補がある場合は選択肢として列挙。調査プロンプトなし、レポート生成プロンプトへ直行
- **Fast Research** — 観点別に最大5個のプロンプトを提示。「以下の N 個で Fast Research をお願いします」とまとめる

**3. レポート生成プロンプト**
調査完了後に貼るプロンプト。目的は「Claude Code が後続分析するための素材」を引き出すこと。人間向けレポートではなく、AI が処理しやすい構造化データを出力させる。**プロンプトの末尾に「結果をここに貼ってください」の案内を含める**。
レポート生成プロンプトには以下を必ず含めること:
- **情報の期間・鮮度** — 各データがいつ時点の情報か明記させる（`データ取得日`、`対象期間`）
- **ソースURL** — 出典元 URL を明記させる。不明な場合はソース名と発行日
- **テーブル形式** — 調査テーマに応じたカラム設計:
  - YouTube: `タイムスタンプ | 発言者 | 主張 | 根拠`
  - PDF/書籍: `ページ | セクション | キーポイント | 詳細`
  - Fast Research: `ソースURL | 発行日 | 主張 | エビデンス | 信頼度`
- **矛盾する情報** — ソース間の食い違いは両論併記し、どちらが新しいか示す
- **欠損データ** — 見つからなかった情報を「データなし」と明示させる
- **要約と未解決の論点** — 最後に追加

## 設計方針
- **短く** — 手順が伝わる最小限の文量にする
- **コードブロック** — プロンプトにはコピーボタンが付く
- **判断を求めない** — 方式は Claude Code が推奨。ユーザーは変えたければ変える
- **レポート生成プロンプトが最重要** — ここで出力品質が決まる
