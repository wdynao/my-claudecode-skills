# Obsidian Vault ノート追加スキル

ユーザーが明示的に保存を指示したとき、知識をObsidian Vaultにノートとして追加する。

## Vault 情報

- パス: `C:\Users\pista\life-vault\`
- 形式: Markdownファイル（.md）
- 構造: フラット。フォルダ階層は気にしない。

## 発動条件

ユーザーが以下のような明示的な指示をした場合のみ発動する:
- 「保存して」「メモして」「ノートにして」
- 「Vaultに追加して」「記録しておいて」
- 「これ覚えておいて」（Vault保存の文脈で）

曖昧な場合は「Vaultに保存しますか？」と確認する。勝手に保存しない。

## ノート作成手順

### 1. 既存ノートを検索する

保存する内容に関連するキーワードで既存ノートを検索する。

```bash
# Vault内をキーワードで検索
grep -r -i -l "キーワード" /c/Users/pista/life-vault/ --include="*.md"

# 複数キーワードで検索して関連ノートを見つける
grep -r -i -l "Obsidian" /c/Users/pista/life-vault/ --include="*.md"
grep -r -i -l "RAG" /c/Users/pista/life-vault/ --include="*.md"
```

検索結果からファイル名を取得し、リンク候補にする。

### 2. ノートを作成する

以下のフォーマットでMarkdownファイルを作成する:

```markdown
---
tags: [タグ1, タグ2]
created: YYYY-MM-DD
source: 情報源（会話、調査、実体験など）
---
# タイトル

ここに洞察・知識の本文を書く。自分の言葉で、1つのアイデアに絞る。

## Links
- [[関連ノート名1]]
- [[関連ノート名2]]
```

### 3. ファイル命名規則

- ファイル名はノートの内容を端的に表す日本語または英語
- スペースはハイフンに置換
- 例: `vertex-ai-チャンク最適化.md`, `obsidian-sync-vs-git.md`

### 4. 保存する

```bash
# ファイルをVaultルートに直接保存
# パス例: C:\Users\pista\life-vault\ノート名.md
```

保存後、ファイルパスとリンクしたノートをユーザーに報告する。

## ノート作成のルール

**1つのノート = 1つのアイデア。** 複数のトピックが含まれる場合は分割する。

**自分の言葉で書く。** 元の文章のコピペではなく、洞察として再構成する。

**リンクを積極的に貼る。** 検索で見つかった関連ノートは `[[ノート名]]` で必ずリンクする。ノートが存在しなくてもリンクを貼ってよい（Obsidianが自動でダングリングリンクとして扱う）。将来そのノートを作ったときに自動で繋がる。

**タグは2-4個。** 多すぎると意味がなくなる。

**sourceを明記する。** どこで得た知識かを記録しておく（例: `Claude会話`, `公式ドキュメント`, `実験結果`）。
