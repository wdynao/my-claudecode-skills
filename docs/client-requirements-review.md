# 広告運用自動化 要件レビュー

**クライアント**: 株式会社スペクトラム 橋本様  
**前提**: スプレッドシート + GAS ベースでの実現を想定

---

## 要件サマリ

| # | 要件 | 実現可否 | 難易度 | 前提条件 |
|---|------|---------|--------|---------|
| ① | 入力→登録用データの自動生成 | ✅ 実現可能 | 低〜中 | マスタシートの整備 |
| ② | データ集計の自動化（API連携） | ⚠️ 条件付き | 中 | APIが使えるプラン契約が必要 |
| ③ | ステータス変更時のChatwork通知 | ✅ 実現可能 | 低 | Chatwork APIトークン、Installable trigger |
| ④ | スプシ間データの集約 | ✅ 実現可能 | 低〜中 | ファイル間のアクセス権限設定 |
| ⑤ | NGワードの自動検出 | ✅ 実現可能 | 低 | NGワードマスタの整備 |
| ⑥ | 書式込みでの転記 | ⚠️ 条件付き | 中〜高 | 同一スプシ内なら容易、別スプシ間は工夫要 |

## 各要件の詳細

### ① 入力→登録用データの自動生成

- **実現可否**: ✅ GASで完全に実現可能
- **実装方法**:
  - onEdit トリガーで入力検知→マスタシートからVLOOKUP的に自動補完
  - URLのドメイン解析・パラメータ付与はJS文字列操作で可能
  - CSV出力は `DriveApp.createFile()` で Google Drive に生成
- **注意点**:
  - simple trigger は30秒上限・外部API呼出不可。URL短縮など外部連携が必要ならinstallable triggerに
  - マスタシートの設計（アフィリエイター名とURL/パラメータの対応表）が品質を左右する
  - onEditのトリガーキューは最大2件、高頻度編集時に取りこぼしの可能性

---

### ② データ集計の自動化（API連携）

- **実現可否**: ⚠️ 条件付きで実現可能
- **実装方法**:
  - アドエビス: API公開済み（RESTful/JSON）。GASの `UrlFetchApp` から呼び出し可能
  - アフィリコード: API公開済み（RESTful/JSON/OAuth認証）。ただしビジネスプラン以上が必要
- **APIが使えない場合の代替**: 手動CSVエクスポート → Google Driveに配置 → GASで読み込み・集計
- **注意点**:
  - アドエビスは対応プラン制限の可能性あり。契約内容を要確認
  - アフィリコードはビジネスプラン以上が条件（スタンダードは拡張申込み要）
  - GASの1日のURLフェッチ上限: 無料2万回/Workspace10万回（日次集計なら十分）
  - 実行時間6分の壁に注意（大量データ時）
- **クライアントへの確認事項**: 現在のアドエビス・アフィリコードの契約プラン

---

### ③ ステータス変更時のChatwork通知

- **実現可否**: ✅ 実現可能（難易度低）
- **実装方法**:
  - Installable trigger の onEdit で変更検知 → Chatwork API v2 にPOST
  - APIトークンは Chatwork管理画面から取得
  - 2025年7月3日以降、パラメータはbodyに含める必要あり（仕様変更注意）
- **注意点**:
  - simple trigger（組み込みonEdit）では `UrlFetchApp` が使えない→必ずInstallable trigger
  - 関数名を「onEdit」以外にする必要あり（重複実行防止）
  - ビジネスプランの場合、APIトークン取得に管理者承認が必要な場合あり
- **工数目安**: GAS経験者なら1〜2時間

---

### ④ スプシ間データの集約

- **実現可否**: ✅ 実現可能
- **実装方法**:
  - `SpreadsheetApp.openById()` で他ファイルを開き、`getValues()` で一括取得
  - JavaScriptの `filter()` で条件抽出（例: 審査日からN日超過）
  - 時間トリガーで定期自動実行（最短1分間隔〜月1回）
- **注意点**:
  - セル単位の `getValue()` ループは極めて遅い→必ず `getValues()` で一括取得
  - `@OnlyCurrentDoc` スコープがあると `openById` が失敗する
  - スクリプト実行者が対象スプレッドシートの閲覧権限を持っていること
  - 実行時間6分上限。大量ファイルの場合はPropertiesServiceで分割実行

---

### ⑤ NGワードの自動検出

- **実現可否**: ✅ 実現可能（難易度低）
- **実装方法**:
  - マスタシートにNGワードリストを管理
  - `includes()`（部分一致）や `RegExp.test()`（正規表現）で照合
  - 該当箇所を `setBackground()` でハイライト、`setNote()` でメモ付与
- **注意点**:
  - スレッド型コメントはGAS標準APIでは非対応（Sheets API v4の高度サービスが必要）
  - 薬機法関連ワードは薬効表現と誇大表現を分離管理すると運用しやすい
  - 正規表現で表記揺れ（「完治」「完全に治る」等）にも対応可能
  - マスタの更新履歴管理（バージョン管理）は別途検討が必要

---

### ⑥ 書式込みでの転記

- **実現可否**: ⚠️ 条件付き
- **実装方法**:
  - 同一スプレッドシート内: `Range.copyTo()` で値＋書式を同時コピー（容易）
  - 異なるスプレッドシート間: 直接コピー不可。以下の回避策が必要
    - 方法A: `Sheet.copyTo()` で一時シートを転記先にコピー → 対象範囲をcopyTo → 一時シート削除
    - 方法B: Sheets API v4 で書式情報をget/setで個別転送（コード量大）
    - 方法C: `getBackground()` / `setBackground()` 等を項目ごとにループ処理
- **注意点**:
  - 異なるスプレッドシート間では「Target range and source range must be on the same spreadsheet」エラー
  - 一時シート方式が最も現実的だが、後処理が必要
  - セル結合の再現には `getMergedRanges()` / `merge()` が別途必要
  - クライアントが「可能であれば」と留保しているのは、この制約を経験的に知っている可能性

## 総合所見

### 実現性まとめ

①③④⑤はGAS+スプシで問題なく実現可能。②はAPI契約プランの確認が先決。⑥は同一スプシ内なら容易だが別スプシ間は要設計。

最終ゴール「最低限の入力→自動生成→CSV出力→インポートのみ人力」は①〜⑤の組み合わせで概ね達成可能。

### GAS共通の制約

- **実行時間上限**: 1回あたり6分
- **トリガー合計上限**: 1日90分
- 大量データ処理や複数ファイル集約を行う場合は、これらの制約を意識した分割設計が必要

### 推奨フェーズ

| フェーズ | 対象要件 | 内容 |
|---------|---------|------|
| Phase 1 | ①⑤ | マスタ整備＋入力自動化＋NGチェック |
| Phase 2 | ③④ | 通知＋集約 |
| Phase 3 | ②⑥ | API連携＋書式転記 |

### クライアントへの確認事項

- [ ] アドエビスの現在の契約プラン（API利用可否）
- [ ] アフィリコードの現在の契約プラン（ビジネスプラン以上か）
- [ ] Chatworkの管理者権限の有無（APIトークン発行に必要）
- [ ] 集約対象のスプレッドシートの数と規模（実行時間見積もりのため）
