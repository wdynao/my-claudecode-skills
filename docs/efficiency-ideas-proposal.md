# 広告運用の業務効率化 — 追加提案アイデア集

既にご依頼いただいている3タスク（A: メンテナンス通知Bot、B: おまとめ率自動計算、C: クリエイティブNGチェックAI）に加えて、ヒアリングと業務分析から浮かび上がった「まだ言語化されていないニーズ」を整理した提案資料です。

---

## Tier 1: 絶対提案すべき（5個）

### 1. パラメータURL生成フォーム（ポカヨケ）

- **Before:** UTMパラメータを手打ち。媒体名・キャンペーン名の表記ゆれがCV計測データに混入
- **After:** Googleフォームに媒体・キャンペーンを選択すると、正規化済みURLが自動生成
- **なぜ刺さるか:** おまとめ率自動計算（タスクB）の精度は元データの品質に依存。インターン生が主な操作者なので、手打ち排除（ポカヨケ）の恩恵が最大
- **実装:** GASのonFormSubmitトリガー。マスタシート（媒体一覧・UTMルール）から自動組み立て
- **複数媒体対応:** Google Ads `{campaignid}` / Meta `{{campaign.id}}` / TikTok `__CAMPAIGN_ID__` の動的マクロ記法の違いをマスタで管理
- **アフィリコードURL発行:** API（Ver.3.12.0〜）で外部からURL登録可能（ビジネスプラン以上）
- **工数:** 低〜中
- **関連する既存タスク:** B（おまとめ率計算）の前段として

### 2. クリエイティブ管理台帳 + 過去成績横断検索

- **Before:** クリエイティブ情報をチェックシート・入稿管理表・報告書に3回手入力。過去の好成績素材を探す手段がない
- **After:** クリエイティブ1点ごとにID付番。1枚の台帳に制作→審査→入稿→成績（CTR/CVR）を集約。「媒体×業種×CTR上位」でフィルタ検索可能
- **なぜ刺さるか:** 既存タスクCのNGチェックAIがGoogleドライブ→スプシに結果を転記する設計。このタイミングでID付番すれば追加工数ほぼゼロで「組織の知識資産」が生まれる
- **実装:** GASでドライブアップロード時に採番（YYYYMMDD+連番）→台帳シートに行追加。QUERY関数で横断検索
- **工数:** 中
- **関連する既存タスク:** C（NGチェックAI）と同一スプシで完結

### 3. KPIアラート（予算消化ペース異常検知 → Chatwork通知）

- **Before:** 月末の予算消化ペースを担当者が感覚で管理。未消化・オーバーに気づくのが遅れる
- **After:** GASの時間トリガーが毎日おまとめ率計算と同時に予算消化率を計算。閾値（例: 残7日で消化率60%未満 or 残3日で110%超）を超えるとChatwork自動通知
- **なぜ刺さるか:** タスクBと同一スクリプトに追加するだけ。Chatwork APIはタスクAで設計済みなのでコピー流用可能。追加工数が最小
- **実装:** 既存タスクBのGASに予算テーブル参照+アラート判定を追加
- **工数:** 低
- **関連する既存タスク:** A（Chatwork通知）+ B（おまとめ率計算）

### 4. CV計測の乖離モニタリング（3ソース突合）

- **Before:** アドエビス・アフィリコード・媒体管理画面のCV数は日常的に乖離するが、誰もモニタリングしていない。計測が壊れているのに気づかず運用判断を誤るリスク
- **After:** 日次で3ソースのCV数を突合し、乖離率が閾値（例: 20%超）を超えるとChatwork通知。「CPAが急に悪化した→実は計測バグだった」事故を防止
- **なぜ刺さるか:** ヒアリングで「CV乖離はビジネス上のペインポイント」と明言あり。全てのKPIアラート・疲弊検知の前提として計測の信頼性担保が不可欠
- **実装:** タスクBのAPI連携を拡張し、媒体APIのCV数も取得。乖離率を計算してアラート
- **工数:** 中（媒体APIの追加接続が必要）
- **関連する既存タスク:** B（おまとめ率計算）の拡張

### 5. 3パターン使い分けルールのドキュメント化 + 判定フォーム

- **Before:** アドエビスのみ/両方/アフィリコードのみの3パターンの使い分けが口伝。インターン生は毎回聞かないと判断できない
- **After:** 判定フローチャートをスプシ1枚にまとめ、クライアント別設定をマスタ化。新規案件登録時にフォームで条件入力→パターン自動判定
- **なぜ刺さるか:** このルールが未ドキュメントのまま自動化を重ねると、GASが間違ったパターンで計算するサイレントバグが発生。タスクBの前提として必要
- **実装:** GAS不要。スプシ1枚のフローチャート+マスタシート。担当者30分ヒアリングで完成
- **工数:** 最小

## Tier 2: 余裕があれば提案（7個）

### 1. クリエイティブ疲弊アラート（Ad Fatigue自動検知）

- 概要: 配信中クリエイティブのCTR・フリークエンシーを日次監視し、疲弊兆候を検知してChatwork通知
- 判定基準: CTR週次10〜15%低下で警戒、25%低下で即対応。フリークエンシー2.5回/週超で要レビュー
- クリエイティブ寿命目安: Meta高予算で14〜21日、Google Displayで28〜35日
- 前提: クリエイティブ管理台帳（Tier1-2）に成績データが蓄積されていること
- 実装難易度: 中

### 2. NGワードリアルタイムハイライト

- 概要: 入稿テキスト入力時にonEditでNGワード照合→該当セルを赤背景
- タスクCのNGチェックAI（Googleドライブ起点）に対する二重防御
- 実装難易度: 低

### 3. 媒体ポリシー変更ウォッチ + 社内自動周知

- 概要: TikTok/Meta等の広告ポリシー改定情報をRSS/スクレイピングで週次取得→Chatwork通知
- NGワードリストの更新トリガーになる（NGチェックの上流工程）
- 実装難易度: 中

### 4. クライアントNGフィードバックのChatwork→スプシ自動転記

- 概要: Chatworkからのフィードバックを自動でNGシートに反映
- タスクCのNGワードリストを自動更新するフローに接続可能
- 実装難易度: 中

### 5. 媒体別入稿フォーマット自動変換

- 概要: 台帳から媒体別仕様（文字数・画像サイズ・フォーマット）のCSVを自動生成
- 実装難易度: 中

### 6. クリエイティブ審査通過率の分析ダッシュボード

- 概要: 制作者別・媒体別のNG率を自動集計。インターン生教育のフィードバックに活用
- 台帳（Tier1-2）完成後にQUERY関数で実現可能
- 実装難易度: 低

### 7. 景表法・薬機法・ステマ規制チェックリストのゲート化

- 概要: 法規制チェックリストの全項目にOKが付かないと承認フローに乗せない仕組み
- 2023年10月施行ステマ規制対応。アフィリエイト広告は消費者庁の重点監視対象
- 実装難易度: 低（スプシのデータ入力規則+GASで実現）

## 提案の全体マップ

既存タスクA/B/Cと今回のTier1提案の依存関係を図示:

```
[タスクA: メンテ通知Bot] ──→ Chatwork API基盤（共用）
                                    ↑
[タスクB: おまとめ率自動計算] ──→ [提案3: KPIアラート]
         ↑                         ↑
  [提案5: 3パターンドキュメント]  [提案4: CV乖離モニタリング]
         ↑
  [提案1: パラメータURL生成フォーム] → 計測品質の土台

[タスクC: NGチェックAI] ──→ [提案2: クリエイティブ管理台帳]
                                    ↓
                          [Tier2: 疲弊アラート / 通過率分析]
```

## 推奨実装順序

1. **提案5（3パターンドキュメント化）** → 工数ゼロ。全自動化の前提条件
2. **提案1（パラメータURL生成フォーム）** → 計測品質の土台。タスクBの精度向上
3. **提案3（KPIアラート）** → タスクBのスクリプトに追加するだけ
4. **提案4（CV乖離モニタリング）** → タスクBの拡張。計測信頼性の担保
5. **提案2（クリエイティブ管理台帳）** → タスクCと同時に構築
